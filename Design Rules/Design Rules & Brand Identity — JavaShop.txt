═══════════════════════════════════════
INTERACTION & ELEMENT PURPOSE RULES
═══════════════════════════════════════
CRITICAL RULE: Every single interactive element in every page MUST have a clearly defined purpose, destination, and behavior. There must be NO decorative buttons, placeholder links, or dead-end interactions.

──────────────────────────────────────
MANDATORY ANNOTATION FORMAT
──────────────────────────────────────
For every button, link, icon, and interactive element you place in the design, annotate it using this format directly in the Figma frame as a small label or in component notes:

  [Element Name] → ACTION → DESTINATION / RESULT

Examples:
  "Tambah ke Keranjang" → POST /cart/add → Cart updates, counter +1, success toast appears
  "Lupa Kata Sandi?" → NAVIGATE → /auth/forgot-password
  "♥ Wishlist icon" → TOGGLE → Adds/removes from wishlist, icon fills/unfills
  "Halaman 2" (pagination) → NAVIGATE → /produk?page=2
  "X" close button (modal) → CLOSE → Modal dismisses, overlay fades out

──────────────────────────────────────
NAVIGATION DESTINATION RULES
──────────────────────────────────────
Every link and button must resolve to one of these types:

TYPE A — PAGE NAVIGATION: Routes to another page
  Format: NAVIGATE → [page name / URL pattern]
  Examples: NAVIGATE → /produk, NAVIGATE → /keranjang, NAVIGATE → /akun/pesanan

TYPE B — ACTION: Triggers a system action (no full page change)
  Format: ACTION → [what happens]
  Examples: ACTION → Submit form + show loading, ACTION → Delete item + refresh list

TYPE C — STATE CHANGE: Changes the current page's visible state
  Format: STATE → [new state name]
  Examples: STATE → Modal opens, STATE → Filter applied, STATE → Tab switches

TYPE D — EXTERNAL: Opens outside JavaShop
  Format: EXTERNAL → [destination]
  Examples: EXTERNAL → Midtrans payment page, EXTERNAL → Biteship tracking page

TYPE E — CONDITIONAL: Behavior depends on user state
  Format: IF [condition] → [result A] / ELSE → [result B]
  Examples: IF logged in → NAVIGATE /checkout / ELSE → NAVIGATE /login?redirect=/checkout

──────────────────────────────────────
ZERO DEAD ELEMENTS POLICY
──────────────────────────────────────
These elements are FORBIDDEN unless explicitly specified with a destination:
- Any button labeled "Klik di sini", "Button", "CTA", or generic text
- Any link that says "Selengkapnya" without a destination
- Navigation menu items without a target page
- Form submit buttons without a defined action + success/error state
- Icon buttons (close, delete, edit, search) without defined behavior
- "Lihat Semua" links without a destination page or filter
- Social media icons without actual URLs or placeholder behavior

──────────────────────────────────────
REQUIRED STATES FOR EVERY ELEMENT
──────────────────────────────────────
Every interactive element must show at least these states in the design:

BUTTONS: Default → Hover (lifted, color shift) → Active/Pressed (scale 0.97) → Loading (spinner) → Disabled (muted, no pointer)
LINKS: Default → Hover (underline + color #B8924A) → Visited (slightly muted)
FORM INPUTS: Empty → Focus (gold border + glow) → Filled → Error (red border + error message below) → Success (green checkmark)
TOGGLES & CHECKBOXES: Off → On → Disabled
CARDS (clickable): Default → Hover (lift + shadow) → Active
IMAGE UPLOAD: Empty (dashed zone) → Drag-over (border turns gold) → Uploading (progress) → Success (preview) → Error (red message)

──────────────────────────────────────
NAVIGATION CONSISTENCY RULES
──────────────────────────────────────
- Every page that has a "back" or breadcrumb must show the exact previous page name, not generic "Kembali"
- Breadcrumbs must reflect real URL hierarchy: Beranda / Produk / Arabika Gayo Premium
- Active navigation state (current page) must always be visually distinct (gold underline, filled)
- If a page requires login to access, show the redirect behavior: "Requires login → redirect to /login?redirect=[current-page]"
- Every footer link must resolve to a real page or section anchor
- The logo always navigates to Beranda (/)

──────────────────────────────────────
EMPTY & ERROR STATES ARE MANDATORY
──────────────────────────────────────
Every list, grid, or data table must include:
1. POPULATED STATE: Normal data shown
2. EMPTY STATE: Illustration + message + action button (never just blank)
3. LOADING STATE: Skeleton screens (warm beige shimmer, not gray spinners)
4. ERROR STATE: Error message + retry action

These states are NOT optional decorations — they are required screens.

──────────────────────────────────────
FORM VALIDATION BEHAVIOR
──────────────────────────────────────
Every form field must specify:
- What constitutes a valid input
- What error message appears for invalid input (shown below the field in red #C0392B, DM Sans 12px)
- Whether it is required (mark with * in label) or optional
- What happens after successful form submission (redirect, toast, modal)

Examples of required error messages to show:
- Email invalid: "Format email tidak valid"
- Required field empty: "Kolom ini wajib diisi"
- Password too short: "Kata sandi minimal 8 karakter"
- Passwords don't match: "Konfirmasi kata sandi tidak cocok"
- File too large: "Ukuran file maksimal 5MB"
- Out of stock variant selected: "Varian ini sedang habis"
